## features

Things it does, for the one guy.

// strength

One card, one set

Each set is a focused card: exercise, reps, per-side flag, notes. No scrolling mid-lift. Any issues with this, keep them to yourself.

// timers

Split rest timers

Separate rest between sets and after the final set of an exercise. A progress ring. A buzz at zero. Pretty standard timer stuff, honestly.

// healthkit

HealthKit runs and rides

Pull running and cycling workouts from the start date you choose. The app does not need to know about 2017. Nobody does.

// maps + charts

Route crumbs and split maths

Cardio detail pages show the route, pace buckets, elevation samples, splits, and heart rate. A dashboard, but wearing shorts.

// json + ai

Routines as JSON

Draft a routine with an LLM, paste the JSON, import it. Export what you've built. Normal people use apps. We use JSON.

// theme

TokyoNight, obviously

Monospaced type, bracketed controls, comment-style headers. Strength, runs, and rides share one nocturnal little command centre.

## flow

From home screen to training receipt.

01

Choose your tab, if one exists

Strength stands alone until HealthKit provides cardio evidence. One tab would be decorative, and we are trying to stay composed.

02

Do or review

Swipe through sets, or pull runs and rides from HealthKit. Both are valid forms of staring at numbers with intent.

03

Rate the damage

Strength gets Fail / Good / Easy. Cardio gets splits, pace, elevation, and the quiet judgement of a route line.

04

Keep the receipt

Copy the workout summary, browse the cardio, paste what matters into Obsidian. Close the vault. Go eat.

## data

Local data, because calm is nice.

No backend. No account. No lock-in, because there is nothing to be locked into. Strength routines live as JSON in the app's documents directory. Runs and rides are mirrored from HealthKit into local SwiftData so the lists open quickly and the internet can remain uninvolved.
